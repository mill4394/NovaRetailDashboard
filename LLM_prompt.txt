You are a senior Python developer and Streamlit dashboard architect.

Build a complete, deployment-ready Streamlit application for NovaRetail and return one executable Python file named app.py.

A. BUSINESS PROBLEM
NovaRetail management needs an interactive customer intelligence dashboard that supports three decisions:
1. Identify the customers generating the most revenue.
2. Identify customer segments that may be at risk based on spending and satisfaction.
3. Identify investment opportunities across product categories, customer regions, and retail channels.

B. DATASET
The dataset is named NR_dataset.xlsx and is stored in the same GitHub repository and directory as app.py.
Read the worksheet named data.

The dataset contains these columns:
- idx
- label
- CustomerID
- TransactionID
- TransactionDate
- ProductCategory
- PurchaseAmount
- CustomerAgeGroup
- CustomerGender
- CustomerRegion
- CustomerSatisfaction
- RetailChannel

Important data notes:
- idx is only an index and should be removed.
- TransactionDate must be converted to datetime.
- PurchaseAmount and CustomerSatisfaction must be numeric.
- One label value may be missing; replace it with Unclassified.
- A TransactionID may appear across multiple rows because one transaction can contain multiple product lines.
- Revenue is the sum of PurchaseAmount.
- Transaction counts must use distinct TransactionID values.
- Average order value must first aggregate PurchaseAmount by TransactionID and then average those transaction totals.
- Purchase frequency should be calculated as distinct transactions divided by distinct customers.

Create a professional dashboard titled:
NovaRetail Customer Intelligence Dashboard

Display four top-level KPI cards:
